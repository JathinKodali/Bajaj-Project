# Quiz Leaderboard System

## Problem Statement
Poll a remote quiz API across 10 sequential rounds, collect event data,
eliminate duplicate entries using a composite key of `roundId` and `participant`,
accumulate scores per participant, and submit the final sorted leaderboard via POST.

## Tech Stack
- **Language:** Java 17
- **JSON Library:** Gson 2.10.1
- **HTTP:** `java.net.http.HttpClient` (built-in)

## Project Structure
```
├── src/
│   └── Main.java          # Application entry point
├── lib/
│   └── gson-2.10.1.jar    # JSON parsing dependency
├── dashboard.html          # Visual replay dashboard (reads poll_data.json)
```

## How to Run

### 1. Set Your Registration Number
Open `src/Main.java` and update `REGISTRATION_NUMBER` with your actual reg number.

### 2. Compile
```bash
javac -cp lib/gson-2.10.1.jar src/Main.java -d out/
```

### 3. Run
```bash
java -cp out:lib/gson-2.10.1.jar Main
```
> **Windows users:** Replace `:` with `;` in the classpath.

### 4. View Dashboard (optional)
After running Main.java, serve the project folder and open `dashboard.html`:
```bash
python3 -m http.server 8080
# Open http://localhost:8080/dashboard.html
```
The dashboard reads `poll_data.json` (auto-generated by Main.java) to replay real API data.

## Approach

1. **Polling** — Sequentially hits the API 10 times (`poll=0` to `poll=9`) with a mandatory 5-second delay between requests.
2. **Deduplication** — Each event is keyed by `roundId|participant`. A `HashSet` ensures no event is counted twice.
3. **Aggregation** — A `HashMap<String, Integer>` accumulates total scores per participant using `merge()`.
4. **Ranking** — Entries sorted in descending order of total score.
5. **Submission** — One single POST to `/quiz/submit` after all polls complete.

## Generated Output Files
- `run_log.txt` — Full execution log
- `result.json` — API submission response (proof)
- `poll_data.json` — Raw poll data for dashboard visualization
